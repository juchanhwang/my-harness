# Inclusive Design

## 1. 포용적 디자인이란?

가능한 한 많은 사람이 사용할 수 있는 제품을 설계하는 것. "평균적 사용자"라는 허구에서 벗어나, 다양한 능력, 상황, 맥락의 사용자를 고려하는 설계 접근법.

Microsoft의 정의: "인간 다양성의 전체 범위를 인식하고 설계하는 것."

### 접근성 vs 포용적 디자인

* **접근성(Accessibility)**: 장애를 가진 사용자를 위한 기술적 표준 (WCAG)
* **포용적 디자인(Inclusive Design)**: 모든 인간 다양성을 고려하는 설계 철학
* 접근성은 포용적 디자인의 **부분집합**

### 장애의 스펙트럼 (Microsoft Inclusive Design Toolkit)

| 유형 | 영구적 | 일시적 | 상황적 |
|------|--------|--------|--------|
| 시각 | 시각 장애 | 백내장 수술 후 | 햇빛 아래 화면 |
| 청각 | 청각 장애 | 귀 감염 | 시끄러운 카페 |
| 운동 | 한 팔 절단 | 팔 깁스 | 아기를 안고 |
| 인지 | 자폐 스펙트럼 | 뇌진탕 | 수면 부족, 스트레스 |
| 언어 | 비발화 | 후두염 | 외국어 환경 |

→ "장애인을 위한 디자인"이 아니라, **모든 사람이 겪을 수 있는 상황을 위한 디자인**

## 2. 색각 이상 (Color Vision Deficiency)

전 세계 남성의 약 8%, 여성의 약 0.5%가 색각 이상.

### 유형

| 유형 | 빈도 | 특징 |
|------|------|------|
| Protanopia (적색맹) | 1.0% 남 | 빨강-초록 구분 불가 |
| Deuteranopia (녹색맹) | 1.0% 남 | 빨강-초록 구분 불가 (가장 흔함) |
| Tritanopia (청색맹) | 0.003% | 파랑-노랑 구분 불가 |
| Achromatopsia (전색맹) | 0.003% | 모든 색 구분 불가 |

### 디자인 가이드라인

1. **색상만으로 정보 전달 금지**
   * 에러: 빨간 보더 + 아이콘 + "에러" 텍스트
   * 성공: 초록 + 체크 아이콘 + "완료" 텍스트
   * 그래프: 색상 + 패턴(빗금, 점) + 직접 라벨링
2. **안전한 색상 조합**
   * 파랑-주황 (대부분의 색각 이상에서 구분 가능) (O)
   * 파랑-빨강 (O)
   * 빨강-초록 (가장 흔한 색각 이상에서 구분 불가) (X)
   * 초록-갈색 (X)
   * 파랑-보라 (X)
3. **충분한 명도 차이**: 색조가 다르더라도 명도가 비슷하면 구분 어려움

### 테스트 도구

* Figma: Stark 플러그인
* Chrome: DevTools → Rendering → Emulate vision deficiencies
* 온라인: Coblis (color-blindness.com/coblis-color-blindness-simulator)

## 3. 인지 장애 (Cognitive Disabilities)

ADHD, 난독증, 자폐 스펙트럼, 기억력 장애, 학습 장애 등. 또한 스트레스, 피로, 노화로 인한 일시적/상황적 인지 부하.

### 디자인 가이드라인

**단순하고 일관된 레이아웃**

* 예측 가능한 네비게이션
* 일관된 UI 패턴 (같은 기능 = 같은 위치, 같은 모양)
* 불필요한 복잡성 제거

**명확한 언어**

* 짧은 문장, 쉬운 단어
* 전문 용어 최소화 (필요시 설명 제공)
* 구체적 지시 ("여기를 클릭" (X) → "저장 버튼을 클릭하세요" (O))

**기억 부하 줄이기**

* 기억에 의존하지 않는 디자인 (재인 > 회상)
* 진행 상태 표시 (지금 어디에 있는지)
* 자동 저장, 실행 취소
* 최근 항목, 자주 사용하는 항목 노출

**주의력 지원**

* 중요한 정보 시각적 강조
* 방해 요소 최소화 (자동 재생 미디어, 깜박임)
* 태스크 중 불필요한 알림 제한
* Progressive disclosure (한 번에 적은 정보)

**에러 예방 & 복구**

* 확인 다이얼로그 (되돌릴 수 없는 액션)
* 관대한 입력 (다양한 형식 허용)
* 명확한 에러 메시지 + 해결 방법
* Undo/Redo 항상 제공

## 4. 난독증 (Dyslexia)

인구의 약 10-15%가 난독증 스펙트럼. 글자가 뒤섞여 보이거나, 읽기 속도가 느림.

### 디자인 가이드라인

* **좌측 정렬**: 양쪽 정렬(justify) 금지 — 불규칙한 단어 간격
* **Sans-serif 폰트**: Arial, Verdana, 또는 Atkinson Hyperlegible, OpenDyslexic
* **적절한 행간**: 1.5 이상
* **적절한 행장**: 60-70자 이내
* **대문자 텍스트 최소화**: 단어 형태(word shape) 인식이 어려움
* **굵은 텍스트 + 색상**으로 강조 (이탤릭은 읽기 어려움)
* **충분한 대비, 순수 흰색 배경 피하기**: 크림색(#F5F5DC) 배경 선호하는 사용자 있음

## 5. 고령자 (Older Adults)

65세 이상 인구가 전 세계적으로 증가. 시력, 운동 능력, 인지 능력 변화.

### 디자인 가이드라인

**시각**

* 큰 기본 폰트 (16px 최소, 18px+ 권장)
* 높은 대비 (AAA 수준 목표)
* 확대 가능한 UI (pinch-to-zoom 허용)

**운동**

* 큰 터치 타겟 (48px+)
* 넉넉한 요소 간 간격
* 복잡한 제스처 대안 (더블 탭, 핀치 → 버튼)
* 떨리는 손을 고려한 관대한 클릭 영역

**인지**

* 단순한 네비게이션 (깊이 최소화)
* 명확하고 쉬운 언어
* 단계별 안내 (한 번에 한 가지)
* 도움말 쉽게 접근

**기술 친숙도**

* 기술 용어 피하기 ("클라우드에 동기화" → "자동 저장")
* 명시적 피드백 ("저장되었습니다" 큰 텍스트)
* 실수 방지 및 복구 용이

## 6. 운동 장애 (Motor Disabilities)

파킨슨, 관절염, 근위축, 또는 일시적/상황적 (깁스, 한 손 사용).

### 디자인 가이드라인

* **큰 클릭/터치 영역**: 44x44px 이상
* **충분한 간격**: 인접 요소 간 최소 8px
* **키보드 완전 지원**: 마우스 없이 모든 기능 사용
* **드래그 앤 드롭 대안**: 항상 키보드/버튼 대안 제공
* **시간 제한 관대**: 충분한 시간 또는 연장 옵션
* **복잡한 제스처 회피**: 멀티터치, 정밀 스와이프 대신 단순 탭

## 7. 국제화 / 다국어 (Internationalization)

### RTL (Right-to-Left) 지원

아랍어, 히브리어 등 RTL 언어:

* 레이아웃 미러링 (로고, 방향 아이콘은 미러링 안 함)
* CSS `direction: rtl` 또는 `dir="rtl"`
* Logical properties 사용: `margin-inline-start` (not `margin-left`)

### 텍스트 확장

번역 시 텍스트 길이 변화 (영어 → 독일어 30-40% 확장):

* 고정 너비 레이아웃 피하기
* Truncation/Ellipsis 대비
* 버튼 텍스트가 길어져도 깨지지 않게

### 문화적 차이

* 색상 의미 (빨강: 서양=위험, 중국=행운)
* 날짜 형식 (MM/DD vs DD/MM)
* 이름 구조 (이름 + 성 vs 성 + 이름)
* 이미지/아이콘의 문화적 적절성

## 8. 포용적 디자인 체크리스트

* [ ] 색상만으로 정보 전달하지 않음
* [ ] 색각 이상 시뮬레이션 테스트
* [ ] 단순하고 일관된 레이아웃
* [ ] 명확한 언어, 짧은 문장
* [ ] 큰 터치 타겟 (44px+)
* [ ] 키보드 완전 지원
* [ ] `prefers-reduced-motion` 지원
* [ ] `prefers-color-scheme` 지원
* [ ] `prefers-contrast` 고려
* [ ] 텍스트 확대 시 레이아웃 유지
* [ ] 시간 제한 관대 또는 연장 가능
* [ ] 에러 예방 및 명확한 복구 경로
