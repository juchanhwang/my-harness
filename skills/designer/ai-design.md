# AI Design

## AI/LLM 시대의 UX 설계

AI 기능이 제품에 통합되면서 새로운 디자인 패턴이 필요하다.

### 핵심 원칙

1. **투명성 (Transparency)** — AI가 생성한 콘텐츠임을 명확히 표시
2. **제어권 (User Control)** — 사용자가 AI 결과를 수정/거부할 수 있어야 함
3. **점진적 신뢰 (Progressive Trust)** — AI 정확도를 점진적으로 보여주며 신뢰 구축
4. **graceful degradation** — AI 실패 시 수동 대안 항상 제공

### Conversational UI 패턴

```
챗봇/어시스턴트 디자인:
- 사용자 의도 파악 → 명확한 확인 질문
- 멀티턴 대화의 컨텍스트 유지
- 빈 상태 (empty state) = 대화 시작 유도
- 에러 = "이해하지 못했습니다" + 대안 제안
- 스트리밍 응답 (타이핑 인디케이터)
```

### AI 기능 UX 패턴

**자동 완성 / 제안:**

* 인라인 제안 (ghost text) — Tab으로 수락
* 제안의 confidence level 시각화
* 쉬운 dismiss (Esc)
* 제안 빈도 조절 (너무 잦으면 방해)

**AI 생성 콘텐츠:**

* 명확한 "AI 생성" 라벨링
* 편집 가능한 상태로 제공
* 원본과 AI 버전 비교 기능
* 재생성 (regenerate) 버튼

**프롬프트 디자인:**

* 빈 입력창보다 예시 프롬프트 제공
* 프롬프트 템플릿 / 프리셋
* 파라미터 조절 UI (톤, 길이 등)
* 히스토리 관리

### 윤리적 고려사항

* **편향 (Bias)**: AI 추천이 특정 그룹에 불리하지 않은지 검증
* **프라이버시**: 사용자 데이터 사용 범위 명시, opt-out 제공
* **의존성**: AI에 과도하게 의존하지 않도록 수동 옵션 유지
* **책임**: AI 결정의 최종 책임은 사용자에게 (인간 in the loop)

### AI 디자인 체크리스트

* [ ] AI 생성 콘텐츠에 라벨이 있는가?
* [ ] 사용자가 결과를 수정할 수 있는가?
* [ ] AI 실패 시 대안이 있는가?
* [ ] 로딩/처리 중 상태가 명확한가?
* [ ] 프라이버시 정책이 명시되어 있는가?
