# Data Visualization

## 1. 차트 선택 가이드

데이터의 성격에 따라 적절한 시각화 유형 선택.

### 비교 (Comparison)

| 차트 | 용도 | 주의 |
|------|------|------|
| Bar Chart (수직) | 카테고리별 값 비교 | 12개 이하 카테고리 |
| Bar Chart (수평) | 긴 라벨, 많은 카테고리 | 라벨 잘림 방지 |
| Grouped Bar | 카테고리 내 하위 비교 | 그룹당 4개 이하 |
| Radar Chart | 다차원 비교 | 5-8개 축. 직관성 낮음 |

### 추이/변화 (Trend)

| 차트 | 용도 | 주의 |
|------|------|------|
| Line Chart | 시간에 따른 연속 변화 | 5개 이하 라인 |
| Area Chart | 변화량 강조, 누적 비교 | 겹치면 Stacked 사용 |
| Sparkline | 인라인 트렌드 (테이블 내) | 레이블 불필요 |

### 구성/비율 (Composition)

| 차트 | 용도 | 주의 |
|------|------|------|
| Pie Chart | 전체 대비 부분 비율 | **5개 이하** 조각. 비율 차이 클 때만 |
| Donut Chart | Pie의 변형, 중앙에 총합 표시 | 같은 제한 |
| Stacked Bar | 카테고리별 구성 비교 | 100% 기준 시 비율 비교 용이 |
| Treemap | 계층적 구성 | 많은 카테고리, 크기로 비교 |

### 분포 (Distribution)

| 차트 | 용도 | 주의 |
|------|------|------|
| Histogram | 연속 데이터 분포 | 적절한 bin 크기 |
| Box Plot | 분포 요약 (중앙값, 사분위) | 통계 리터러시 필요 |
| Scatter Plot | 두 변수 간 관계/분포 | 데이터 포인트 많을 때 |

### 관계 (Relationship)

| 차트 | 용도 | 주의 |
|------|------|------|
| Scatter Plot | 두 변수 상관관계 | 트렌드라인 추가 |
| Bubble Chart | 3변수 관계 (x, y, 크기) | 크기 비교 직관성 낮음 |
| Heatmap | 2차원 매트릭스 값 | 색상 스케일 명확히 |

### 차트 선택 원칙

1. **단순한 것이 최선**: Pie보다 Bar가 비교에 더 정확
2. **3D 차트 절대 금지**: 왜곡, 가독성 저하
3. **Dual axis 최소화**: 두 개의 y축은 혼란. 가능하면 분리된 차트로

## 2. 데이터-잉크 비율 (Data-Ink Ratio)

Edward Tufte의 핵심 원칙: "데이터를 표현하는 잉크의 비율을 최대화하라".

```
Data-Ink Ratio = Data Ink / Total Ink
```

### 제거할 것 (Chartjunk)

* 3D 효과, 그림자, 그라디언트 배경
* 불필요한 격자선 (gridlines) — 연한 색으로 최소화
* 장식적 요소 (클립아트, 로고)
* 불필요한 범례 (직접 라벨링이 가능하면)
* 두꺼운 테두리, 과도한 축 라인

### 남길 것

* 데이터 포인트
* 최소한의 축 라벨
* 직접 라벨링 (범례 대신 데이터 옆에 직접)
* 의미 있는 격자선 (사선 비교에 필요한 경우)

### Before/After

```
Before (낮은 비율):
┌──────────────────────────────┐
│ Sales Report                 │
│ ┌──┬──┬──┬──┬──┬──┬──┬──┐   │
│ │░░│██│░░│██│░░│██│░░│██│   │
│ │░░│██│░░│██│░░│██│░░│██│   │
│ └──┴──┴──┴──┴──┴──┴──┴──┘   │
│ [Legend] ■ Q1 ░ Q2          │
│ Source: Company Data 2024    │
└──────────────────────────────┘

After (높은 비율):
Q1  ████████████  $120K
Q2  ██████████    $100K
Q3  ██████████████ $140K
Q4  ████████████████ $160K
```

## 3. 대시보드 디자인

### 레이아웃 원칙

**역피라미드**: 가장 중요한 KPI를 상단에, 상세 데이터를 하단에.

```
┌─────────────────────────────────────┐
│  KPI 1  │  KPI 2  │  KPI 3  │ KPI 4│  ← Summary Cards
├─────────────────────┬───────────────┤
│                     │               │
│  Primary Chart      │  Secondary    │  ← Main Insight
│                     │  Chart        │
├─────────┬───────────┴───────────────┤
│         │                           │
│ Filter  │  Detail Table             │  ← Detailed Data
│         │                           │
└─────────┴───────────────────────────┘
```

### 대시보드 유형

**운영 (Operational)**: 실시간 모니터링. 높은 갱신 빈도. 명확한 임계값/알림.

**분석 (Analytical)**: 트렌드/패턴 탐색. 필터/드릴다운 제공. 비교 분석.

**전략 (Strategic)**: 고수준 KPI. 목표 대비 현황. 경영진용. 단순 명료.

### KPI 카드 디자인

```
┌─────────────────┐
│ Monthly Revenue  │  ← 지표명
│ $142,500        │  ← 현재 값 (크고 볼드)
│ ▲ 12.5%         │  ← 변화율 (색상 + 방향)
│ vs last month   │  ← 비교 기준
└─────────────────┘
```

### 필터 & 컨트롤

* 전역 필터 (날짜 범위, 세그먼트)는 상단에
* 차트별 필터는 해당 차트 근처에
* 필터 적용 상태 명확히 표시 (칩, 태그)
* 필터 초기화 버튼 항상 제공

## 4. 색상 전략

### Sequential (순차적)

같은 색조의 밝은→어두운. 양적 데이터 (매출 범위, 온도).

```
#e0f3ff → #b0d4ff → #6ea8fe → #3b82f6 → #1e40af
```

### Diverging (발산형)

중간점에서 양극으로. 긍정/부정, 기준 대비 (수익/손실).

```
#ef4444 → #fca5a5 → #f3f4f6 → #93c5fd → #3b82f6
(부정)                (중립)               (긍정)
```

### Categorical (범주형)

구별 가능한 다양한 색상. 카테고리, 그룹.

* 최대 8-10개 (그 이상은 구별 어려움)
* 색각 이상 안전 팔레트 사용 (colorbrewer2.org)

### 강조

* 하나의 데이터 포인트 강조: 해당 포인트만 색상, 나머지 회색
* 임계값 이상/이하 강조: 빨강/초록 경계선

## 5. 인터랙티브 시각화

### 필수 인터랙션

* **Tooltip**: 호버 시 정확한 값 표시 (가장 기본)
* **Zoom/Pan**: 시계열 데이터 탐색
* **Filter**: 범주 선택/해제
* **Drill-down**: 요약 → 상세 데이터
* **Cross-filtering**: 하나의 차트 선택이 다른 차트를 필터링

### 반응형 차트

* 모바일: 단순화. 툴팁 대신 탭으로 데이터 표시
* 축 라벨 회전 또는 생략
* 범례를 차트 아래로 이동
* 스와이프로 시간 범위 변경

## 6. 접근성

### 차트 접근성 체크리스트

* [ ] 색상만으로 데이터 구분하지 않음 (패턴, 라벨 병행)
* [ ] 충분한 색상 대비 (인접 영역 간 3:1 이상)
* [ ] 텍스트 대안 제공 (aria-label, 데이터 테이블)
* [ ] 키보드로 데이터 포인트 탐색 가능
* [ ] 스크린 리더용 차트 설명 제공
* [ ] 색각 이상 시뮬레이션 테스트

### 데이터 테이블 대안

모든 차트 아래에 접근 가능한 데이터 테이블을 숨겨둔다:

```html
<details>
  <summary>데이터 테이블 보기</summary>
  <table>...</table>
</details>
```

## 7. 안티패턴

* **Pie chart with 10+ slices**: 구별 불가. Bar chart 사용
* **Truncated y-axis**: 0에서 시작하지 않으면 차이 과장 (Bar chart에서 특히)
* **Rainbow color maps**: 인지적으로 왜곡. Sequential 사용
* **Animated chart on load**: 느리고 정보 전달 방해
* **Dashboard에 모든 데이터**: 정보 과부하. "So what?" 테스트
