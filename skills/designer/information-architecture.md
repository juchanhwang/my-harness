# Information Architecture

## 1. IA란?

정보 구조(Information Architecture)는 콘텐츠를 조직하고, 라벨링하고, 사용자가 원하는 것을 찾을 수 있게 구조화하는 실천. Peter Morville과 Louis Rosenfeld가 정립한 "정보 구조의 3원" — 사용자, 콘텐츠, 맥락의 교차점.

---

## 2. IA 핵심 원칙

### Dan Brown의 8대 IA 원칙

1. **Objects 원칙**: 콘텐츠를 생명력 있는 객체로 취급. 타입, 속성, 행동을 가진 엔티티.
2. **Choices 원칙**: 의미 있는 선택지를 제공하되, 과도하지 않게. Hick's Law — 선택지가 많을수록 결정 시간 증가.
3. **Disclosure 원칙**: 다음 단계를 결정하기에 충분한 정보만 미리 보여준다. Progressive disclosure.
4. **Exemplars 원칙**: 카테고리의 내용을 설명할 때 예시를 보여준다. "전자기기" 옆에 "노트북, 스마트폰, 태블릿" 표시.
5. **Front Doors 원칙**: 사용자의 50%+ 는 홈페이지가 아닌 다른 곳에서 진입. 모든 페이지가 독립적으로 이해 가능해야.
6. **Multiple Classification 원칙**: 같은 콘텐츠에 접근하는 다양한 경로 제공. 검색, 네비게이션, 태그, 최근 항목.
7. **Focused Navigation 원칙**: 하나의 네비게이션 체계는 하나의 목적. 혼합하지 않는다.
8. **Growth 원칙**: 콘텐츠가 10배 증가해도 구조가 유지되어야 한다. 확장성 설계.

## 3. 네비게이션 패턴

### 글로벌 네비게이션 (Global Navigation)

모든 페이지에서 접근 가능한 최상위 네비게이션.

* **수평 탑 바**: 웹 표준. 5-7개 항목 (Mega menu로 확장)
* **사이드바**: SaaS/앱. 8-15개 항목. 아이콘 + 라벨. 접이식(collapsible)
* **하단 탭**: 모바일. 3-5개 핵심 목적지

### 로컬 네비게이션 (Local Navigation)

특정 섹션 내 하위 이동.

* **탭**: 같은 레벨의 콘텐츠 전환
* **사이드 메뉴**: 섹션 내 하위 페이지
* **Breadcrumb**: 현재 위치와 상위 경로 표시

### 컨텍스트 네비게이션 (Contextual Navigation)

콘텐츠 내에서 자연스럽게 연결.

* **인라인 링크**: 텍스트 내 관련 페이지 연결
* **관련 콘텐츠**: "이것도 좋아할 수 있어요"
* **다음/이전**: 순서가 있는 콘텐츠

### 네비게이션 선택 기준

| 기준 | 수평 탑바 | 사이드바 | 하단 탭 |
|------|----------|---------|--------|
| 항목 수 | 5-7 | 8-20 | 3-5 |
| 깊이 | 얕은(1-2) | 깊은(2-4) | 얕은(1) |
| 플랫폼 | 웹 | 웹앱/SaaS | 모바일 |
| 콘텐츠 공간 | 최대 | 줄어듦 | 최대 |
| 발견성 | 높음 | 높음 | 높음 |

## 4. 사이트맵 (Sitemap)

### 구조 유형

**계층형 (Hierarchical)** 가장 일반적. 트리 구조. 상위 → 하위.

```
Home
├── Products
│   ├── Category A
│   └── Category B
├── About
│   ├── Team
│   └── Mission
├── Blog
└── Contact
```

**플랫 (Flat)** 깊이 최소화. 모든 페이지가 1-2클릭 내.

* 소규모 사이트, 대시보드

**허브 앤 스포크 (Hub & Spoke)** 중앙 허브에서 독립적 섹션으로 분기.

* 모바일 앱의 일반적 패턴
* iOS 설정 앱 구조

**매트릭스 (Matrix)** 다차원 필터로 콘텐츠 탐색.

* 이커머스 (카테고리 x 브랜드 x 가격)

### 사이트맵 작성 팁

* 사용자 태스크 기반으로 구조화 (조직 구조 미러링 (X))
* 3클릭 규칙보다 **정보 향기(information scent)**가 중요
* 카드 소팅 결과 반영
* 핵심 페이지에서 시작, 점진적 확장

## 5. 멘탈 모델 (Mental Model)

사용자가 시스템이 어떻게 작동하는지에 대해 가진 내적 표상.

### 왜 중요한가?

* 사용자의 멘탈 모델 =/= 시스템의 실제 구조 → **혼란 발생**
* 디자인의 목표: 시스템을 사용자의 멘탈 모델에 맞추는 것

### 멘탈 모델 파악 방법

* **인터뷰**: "이메일을 보낸다고 상상해보세요. 어떤 단계로 진행되나요?"
* **카드 소팅**: 사용자가 정보를 어떻게 그룹핑하는지
* **유사 제품 분석**: 사용자가 이미 익숙한 패턴
* **검색 쿼리 분석**: 사용자가 사용하는 용어

### 디자인 적용

* **익숙한 패턴 활용**: 장바구니, 검색바, 네비게이션 등 컨벤션 준수
* **라벨링**: 사용자의 언어 사용 (내부 전문 용어 (X))
* **개념적 모델(Conceptual Model)**: 사용자에게 시스템의 작동 원리를 시각적으로 제시

## 6. 라벨링 (Labeling)

### 좋은 라벨의 특성

* **명확**: 한 가지 의미만 전달
* **상호 배타적**: 카테고리 간 겹침 없음
* **사용자 언어**: 조직 내부 용어가 아닌 사용자의 언어
* **일관적**: 명사 vs 동사 통일 ("설정" vs "설정하기" 혼재 (X))

### 라벨 테스트

* **트리 테스트**: "이 라벨을 보고 원하는 정보를 찾을 수 있는가?"
* **First-click Test**: "이 화면에서 첫 번째로 클릭할 곳은?"
* **Highlighter Test**: 사용자가 중요하다고 생각하는 라벨 표시

## 7. 검색 (Search)

### 검색 UX 원칙

* **눈에 띄는 위치**: 상단 중앙 또는 우상단
* **검색 아이콘 + 입력 필드**: 아이콘만 (X) → 아이콘 + 필드 (O)
* **자동완성(Autocomplete)**: 인기 검색어, 최근 검색어
* **필터/정렬**: 검색 결과 정제
* **빈 결과 처리**: "검색 결과가 없습니다" + 제안 (오타 수정, 유사 검색어)

### 검색 vs 브라우징

* 사용자가 무엇을 원하는지 알 때 → 검색
* 탐색/발견이 목적일 때 → 브라우징 (네비게이션, 필터)
* 두 가지 다 제공하는 것이 최선

---

## 8. 안티패턴

* **조직 구조 = 사이트 구조**: 사용자는 내부 팀 구조를 모른다
* **너무 깊은 구조**: 5단계 이상 깊이 → 사용자 길 잃음
* **모호한 라벨**: "솔루션", "리소스" (무엇이 있는지 알 수 없음)
* **과도한 메뉴 항목**: Mega menu에 50개+ 링크
* **Breadcrumb 누락**: 깊은 구조에서 현재 위치 파악 불가
* **일관성 없는 네비게이션**: 페이지마다 다른 네비게이션 구조
