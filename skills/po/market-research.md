# Market Research

## 시장 규모 분석: TAM / SAM / SOM

시장 규모는 투자 유치, 전략 수립, Go-to-Market의 기반이 된다.

```
┌─────────────────────────────────┐
│             TAM                 │  Total Addressable Market
│    ┌───────────────────┐        │  전체 시장 규모
│    │       SAM         │        │
│    │   ┌──────────┐    │        │  Serviceable Addressable Market
│    │   │   SOM    │    │        │  접근 가능 시장
│    │   │          │    │        │
│    │   └──────────┘    │        │  Serviceable Obtainable Market
│    └───────────────────┘        │  실제 획득 가능 시장
└─────────────────────────────────┘
```

### TAM (Total Addressable Market)

* **전체 시장**: 모든 잠재 고객이 우리 제품을 사용할 경우의 시장 규모
* 계산법:
  * **Top-down**: 시장 리포트 기반 (Gartner, Statista 등)
  * **Bottom-up**: (잠재 고객 수) × (고객당 연간 지출) — 더 정확
  * **Value theory**: (해결하는 문제의 경제적 가치) × (해당 문제를 가진 인구)

### SAM (Serviceable Addressable Market)

* **접근 가능 시장**: 지역, 세그먼트, 채널 등으로 필터링한 현실적 시장
* TAM에서 **우리가 실제로 도달할 수 있는** 부분
* 예: TAM이 글로벌이라면, SAM은 한국 + 일본 시장

### SOM (Serviceable Obtainable Market)

* **실제 획득 가능 시장**: 단기(1-3년)에 실제로 확보할 수 있는 시장
* 경쟁, 리소스, 브랜드 인지도를 고려한 현실적 수치
* 보통 SAM의 1-5% (초기 스타트업)

### 실전 예시

```
제품: B2B SaaS 프로젝트 관리 도구

TAM: 글로벌 프로젝트 관리 SW 시장 = $7.5B (2025)
SAM: 아시아-태평양 10-500인 기업 = $1.2B
SOM: 한국 스타트업 (50-200인) = $45M (연간)

계산:
  한국 50-200인 스타트업 수: ~15,000개
  전환 가능 비율: 20% = 3,000개
  ARPU: $15,000/년
  SOM = 3,000 × $15,000 = $45M
```

## 경쟁사 분석 프레임워크

### 경쟁사 유형 분류

| 유형             | 설명           | 예시                    |
| -------------- | ------------ | --------------------- |
| **Direct**     | 같은 문제, 같은 방식 | Notion vs Coda        |
| **Indirect**   | 같은 문제, 다른 방식 | Notion vs Excel+Email |
| **Potential**  | 인접 시장의 강자    | Salesforce → PM 진출    |
| **Substitute** | 근본적 대체재      | PM 도구 vs 화이트보드        |

### 경쟁사 분석 매트릭스

```markdown
| 기준 | 우리 | 경쟁A | 경쟁B | 경쟁C |
|------|------|------|------|------|
| 핵심 타겟 | | | | |
| 가격대 | | | | |
| 핵심 기능 | | | | |
| 차별화 포인트 | | | | |
| GTM 채널 | | | | |
| 약점 | | | | |
| 시장 점유율 | | | | |
| 최근 동향 | | | | |
```

### 경쟁 정보 수집 방법

1. **공개 자료**: 보도자료, 블로그, 채용 공고 (로드맵 힌트), 재무제표
2. **제품 분석**: 직접 사용, 스크린샷 정리, 기능 매핑
3. **고객 피드백**: G2, Capterra, Reddit, 커뮤니티
4. **전환 고객 인터뷰**: 경쟁사에서 우리로 온 고객 / 우리에서 떠난 고객
5. **Industry reports**: Gartner, Forrester, CB Insights
6. **소셜 리스닝**: Twitter/X, LinkedIn, 관련 Slack 커뮤니티

## 트렌드 분석

### PESTEL 프레임워크

외부 환경을 체계적으로 분석:

| 요소                | 분석 대상                 |
| ----------------- | --------------------- |
| **P**olitical     | 규제, 정부 정책, 무역 환경      |
| **E**conomic      | 경기, 금리, 환율, 소비 패턴     |
| **S**ocial        | 인구 변화, 문화 트렌드, 가치관    |
| **T**echnological | 기술 발전, 디지털 전환, AI/자동화 |
| **E**nvironmental | 지속가능성, 탄소 중립, ESG     |
| **L**egal         | 개인정보보호법, 노동법, 지적재산권   |

### 트렌드 해석 프레임워크

```
관찰 (Signal) → 트렌드 (Pattern) → 인사이트 (Meaning) → 기회 (Opportunity)

예시:
Signal: AI 코딩 도구 사용 급증
Pattern: 개발자 생산성 도구에 AI 통합이 표준화
Meaning: 단순 코딩은 AI가 대체, 개발자 역할이 아키텍처/설계로 이동
Opportunity: AI 네이티브 개발 워크플로우 도구
```

## Customer Segment 정의

### Segmentation 기준

| B2C               | B2B               |
| ----------------- | ----------------- |
| 인구통계 (나이, 성별, 소득) | 기업 규모 (직원 수, 매출)  |
| 행동 (사용 빈도, 구매 패턴) | 산업 (SaaS, 금융, 제조) |
| 심리 (가치관, 라이프스타일)  | 기술 성숙도            |
| 지역                | 의사결정 구조           |

### Ideal Customer Profile (ICP) 템플릿

**B2B ICP:**

```markdown
## ICP: [이름]

### 기업 특성
- 규모: [직원 수, 매출]
- 산업: [업종]
- 성장 단계: [Seed/Series A/B/Growth]
- 기술 스택: [관련 도구]

### 바이어 특성
- 직책: [VP Engineering, CTO 등]
- 목표: [달성하려는 것]
- 핵심 페인: [해결하려는 문제]
- 예산 권한: [결정 가능 금액]

### 구매 트리거
- [언제 구매를 고려하기 시작하는가?]

### 성공 기준
- [어떻게 되면 성공인가?]

### 경고 신호 (Non-ICP)
- [이런 고객은 피한다]
```

## Positioning Statement 작성

### Geoffrey Moore 포지셔닝 템플릿

```
For [target customers]
Who are dissatisfied with [current alternative]
Our product is a [new category]
That provides [key problem-solving capability]
Unlike [the product alternative]
We have assembled [key whole product features for your specific application]
```

### Positioning Canvas

```
┌─────────────────────────────────────────┐
│ WHO: 타겟 고객                           │
│ → 가장 강한 페인을 가진 세그먼트           │
├─────────────────────────────────────────┤
│ WHAT: 제품 카테고리                       │
│ → 고객이 이해하는 기존 카테고리 or 새 카테고리│
├─────────────────────────────────────────┤
│ WHY: 차별화된 가치                        │
│ → 경쟁사가 쉽게 따라할 수 없는 고유 가치    │
├─────────────────────────────────────────┤
│ PROOF: 증거                              │
│ → 차별화를 뒷받침하는 데이터/사례           │
└─────────────────────────────────────────┘
```

### 포지셔닝 검증 체크리스트

* [ ] 타겟 고객이 즉시 이해하는가?
* [ ] 경쟁사와 명확히 구분되는가?
* [ ] 실제 고객 가치에 기반하는가? (내부 관점이 아닌)
* [ ] 한 문장으로 전달 가능한가?
* [ ] 시간이 지나도 유효한가? (일시적 기능이 아닌 근본적 가치)

### 포지셔닝 실패 패턴

1. **너무 넓음**: "모든 사람을 위한 도구" → 아무도 위한 도구가 아님
2. **기능 기반**: "100개 기능이 있습니다" → 고객은 기능이 아니라 결과를 원함
3. **경쟁 기반**: "X보다 낫다" → 경쟁사가 변하면 포지셔닝도 무너짐
4. **내부 관점**: "우리 기술이 대단하다" → 고객은 기술이 아니라 가치를 산다
