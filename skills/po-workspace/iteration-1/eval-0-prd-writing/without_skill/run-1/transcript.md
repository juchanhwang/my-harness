## 접근 방법

일반적인 제품 관리(Product Management) 지식을 활용하여 PRD를 작성했다. 별도의 스킬 파일이나 외부 참고 자료 없이 아래 순서로 진행했다.

1. **문제 정의**: 제공된 데이터(전환율 30% vs 벤치마크 50%)를 기반으로 비즈니스 임팩트를 정량화했다.
2. **페르소나 구성**: 10~50명 엔지니어링 팀 매니저의 전형적인 Pain Point, 목표, 행동 패턴을 정리했다.
3. **솔루션 구조화**: 온보딩 개선을 위한 5개 핵심 Feature를 구성했다 (환영 플로우, 체크리스트, 템플릿, 코치마크, 이메일 드립).
4. **PRD 섹션 선정**: 팀 공유용 실무 PRD에 필요한 섹션을 포함했다 — 배경, 목표/지표, 페르소나, 솔루션, 기술 요구사항, 타임라인, 리스크, 오픈 이슈 등.
5. **가정 명시**: 실제 데이터 없이 작성한 항목은 `[가정]` 레이블로 명시했다.
