# 탐색적 테스트 (Exploratory Testing)

## 탐색적 테스트란

사전에 정해진 스크립트 없이, 테스터의 **경험, 직관, 호기심**을 활용해 시스템을 탐험하는 테스트 방법. 자동화 테스트가 "예상된 것이 동작하는지" 확인한다면, 탐색적 테스트는 **"예상하지 못한 것을 발견"**한다.

## 탐색적 테스트 ≠ 즉흥 테스트

|       | 탐색적 테스트         | 즉흥 테스트 (Ad-hoc) |
| ----- | --------------- | --------------- |
| 목적    | 명확한 탐색 목표       | 없음              |
| 시간 제한 | 타임박싱            | 없음              |
| 기록    | 세션 노트           | 없거나 부실          |
| 학습    | 시스템 이해 → 다음 테스트 | 학습 미축적          |
| 재현 가능 | 세션 기록으로 가능      | 어려움             |

## 세션 기반 테스트 관리 (SBTM)

### 세션 차터 (Session Charter)

탐색적 테스트의 나침반. 무엇을, 왜, 어떻게 탐색할지 정의.

```markdown
## 세션 차터

**탐색 대상**: 결제 프로세스 (카드 결제)
**목적**: 결제 실패 시 사용자 경험 검증
**시간**: 45분
**테스터**: Hawkeye

### 탐색 아이디어
- 유효하지 않은 카드 번호로 시도
- 결제 중 네트워크 끊기 (비행기 모드)
- 결제 중 뒤로가기 / 새로고침
- 매우 작은 금액 (1원) / 매우 큰 금액
- 만료된 카드 사용
- 연속 빠른 결제 시도 (더블 클릭)
- 결제 진행 중 같은 세션에서 다른 탭 열기
```

### 세션 노트

```markdown
## 세션 노트

**차터**: 결제 프로세스 탐색
**시간**: 14:00-14:45 (45분)
**테스터**: Hawkeye
**빌드**: v2.3.1-rc1

### 발견 사항

🐛 **버그**:
1. [S2] 결제 버튼 더블 클릭 시 두 번 결제됨 — 디바운스 없음
2. [S3] 만료된 카드 에러 메시지가 영어로 표시 ("Card expired")

❓ **질문**:
1. 결제 실패 후 재시도 제한이 있어야 하지 않나?
2. 부분 결제 (잔액 부족 시) 지원 예정?

💡 **아이디어**:
1. 결제 완료까지 버튼 비활성화 필요
2. 결제 진행 중 페이지 이탈 시 확인 모달 필요

📊 **테스트 비율**:
- 탐색: 70% | 셋업: 15% | 버그 리포팅: 15%
```

## 탐색적 테스트 휴리스틱

### SFDPOT (San Francisco Depot)

James Bach의 품질 속성 휴리스틱:

| 약자             | 속성      | 탐색 질문                |
| -------------- | ------- | -------------------- |
| **S**tructure  | 코드/아키텍처 | 어떤 모듈이 가장 복잡한가?      |
| **F**unction   | 기능      | 이 기능이 모든 조건에서 동작하는가? |
| **D**ata       | 데이터     | 빈 데이터, 대용량, 특수문자는?   |
| **P**latform   | 플랫폼     | 브라우저, OS, 디바이스 차이는?  |
| **O**perations | 운영      | 장시간 사용, 업데이트, 백업은?   |
| **T**ime       | 시간      | 동시성, 타임존, 만료, 자정은?   |

Michael Bolton의 일관성 휴리스틱:

| --------------------- | --------- | --------------------- |
| **F**amiliarity       | 유사 제품과 비교 | 경쟁 제품은 이걸 어떻게 처리?     |

| **I**mage             | 브랜드/이미지   | 이 동작이 회사 이미지에 맞나?     |

| **C**laims            | 문서/요구사항   | 문서에 적힌 대로 동작하나?       |

| **P**roduct           | 제품 내부     | 다른 기능과 상호작용은?         |
| **S**tandards         | 표준        | 접근성, 보안, 업계 표준은?      |

## 인지 편향 활용

QA 엔지니어가 알아야 할 인지 편향과 이를 **역으로 활용**하는 방법:

### 확증 편향 (Confirmation Bias)

"작동할 거야"라고 생각하면 작동하는 경우만 봄. → **역활용**: 의도적으로 "어떻게 하면 깨뜨릴 수 있을까?"만 생각하는 세션.

### 앵커링 (Anchoring)

첫 번째 정보에 고정됨. → **역활용**: 매번 다른 시작점에서 탐색. 어제 로그인부터 했으면 오늘은 검색부터.

### 가용성 편향 (Availability Bias)

최근 경험한 버그 유형만 찾으려 함. → **역활용**: 과거 버그 데이터베이스를 분석해 다양한 유형을 체계적으로 탐색.

### 정상화 편향 (Normalization of Deviance)

"원래 그런 거야"라며 이상 징후를 무시. → **역활용**: "처음 보는 사람이라면 이게 정상이라고 생각할까?" 자문.

## 탐색적 테스트 기법

### 1. 투어 (Tours)

목적을 가지고 시스템을 여행하는 방식.

| 투어         | 방법                 |
| ---------- | ------------------ |
| 가이드 투어     | 사용자 매뉴얼대로만 따라가기    |
| 무법자 투어     | 허용되지 않은 모든 것 시도    |
| 쓰레기 수집가 투어 | 가장 안 쓰는 기능 탐색      |
| 랜드마크 투어    | 핵심 기능만 빠르게 순회      |
| 슈퍼 모델 투어   | UI만 집중 (레이아웃, 일관성) |
| 역사가 투어     | 이전 버전과 비교          |

### 2. 페르소나 기반 탐색

다양한 사용자 유형으로 시스템 탐색.

```markdown
페르소나 목록:
- 🧓 IT 비전문가 (60대, 첫 사용)
- 🏃 바쁜 사용자 (빠르게 클릭, 기다리지 않음)
- 😈 악의적 사용자 (해킹 시도)
- 🌍 국제 사용자 (한국어 외 환경)
- ♿ 장애인 사용자 (키보드만, 스크린리더)
- 📱 모바일 사용자 (3G, 작은 화면)
```

### 3. 시나리오 변형

해피 패스를 기반으로 변형을 만들어 탐색.

```
기본: 로그인 → 상품 선택 → 장바구니 → 결제

변형들:
- 로그인 → 상품 선택 → [뒤로가기] → 다른 상품 → 장바구니 → 결제
- 로그인 → 상품 선택 → 장바구니 → [브라우저 닫기] → [재접속] → 장바구니 유지?
- 로그인 → 상품 100개 → 장바구니 → 결제
- [로그인 안 함] → 상품 선택 → 장바구니 → 결제 시도 → 로그인 유도?
```

## 탐색적 테스트 리포트

```markdown
## 탐색적 테스트 요약

**기간**: 2024-01-15 ~ 2024-01-16
**테스터**: Hawkeye
**총 세션**: 4개 (180분)
**빌드**: v2.3.1-rc1

### 세션 결과
| 세션 | 차터 | 시간 | 버그 | 질문 |
|------|------|------|------|------|
| 1 | 결제 프로세스 | 45분 | 2 | 2 |
| 2 | 사용자 프로필 | 45분 | 1 | 1 |
| 3 | 검색 기능 | 45분 | 3 | 0 |
| 4 | 모바일 반응형 | 45분 | 2 | 1 |

### 발견 요약
- 총 버그: 8개 (S2: 1, S3: 5, S4: 2)
- 총 질문: 4개
- 총 아이디어: 6개

### 가장 리스크 높은 영역
1. 결제 — 더블 클릭 방지 없음
2. 검색 — XSS 가능성 있는 입력 처리
```

## 체크리스트

* [ ] 탐색 세션에 명확한 차터가 있는가
* [ ] 타임박싱(30-90분)이 되어 있는가
* [ ] 세션 노트를 기록하고 있는가
* [ ] 다양한 휴리스틱을 활용하고 있는가
* [ ] 발견 사항이 적절히 분류되는가 (버그/질문/아이디어)
* [ ] 자동화 테스트와 보완적으로 사용되는가
* [ ] 세션 결과가 팀에 공유되는가
